Overview

SendGrid is an industry-leading email delivery platform that provides reliable email sending with excellent deliverability rates. This guide will walk you through setting up SendGrid for your Help Desk application.

What you'll need:
  • A SendGrid account (free tier available)
  • Access to your domain's DNS settings
  • Firebase CLI installed on your computer

1. Create a SendGrid Account

1 Go to sendgrid.com and click Start for Free

2 Complete the registration process with your email and company information

3 Verify your email address by clicking the link SendGrid sends you

Free Tier: SendGrid offers 100 emails/day free forever, which is sufficient for most small help desks.

2. Verify Your Domain

Domain verification improves email deliverability and prevents your emails from going to spam.

1 In the SendGrid dashboard, go to Settings → Sender Authentication

2 Click Authenticate Your Domain

3 Select your DNS host (or choose "Other" if not listed)

4 Enter your domain name (e.g., mail.yourdomain.com)

5 SendGrid will provide DNS records to add. You'll need to add these to your domain's DNS settings:

Record Type Purpose Example
CNAME DKIM Authentication (3 records) s1._domainkey.yourdomain.com
CNAME Return Path em1234.yourdomain.com

6 After adding the DNS records, click Verify in SendGrid

Note: DNS changes can take up to 48 hours to propagate, though usually it's much faster (15-30 minutes).

3. Create an API Key

1 In the SendGrid dashboard, go to Settings → API Keys

2 Click Create API Key

3 Give it a name like HelpDesk-Production

4 Select Full Access or choose Restricted Access with these permissions:

  • Mail Send: Full Access
  • Inbound Parse: Full Access (if using email replies)

5 Click Create & View

Important: Copy and save your API key now! SendGrid will only show it once. It looks like: SG.xxxxxxxxxxxxxxxxxxxxxxxx

4. Configure Firebase Secrets

Now you need to store your SendGrid API key as a Firebase secret.

1 Open a terminal and navigate to your project's functions directory

2 Run the following command:

firebase functions:secrets:set SENDGRID_API_KEY

3 When prompted, paste your SendGrid API key

4 Deploy your functions to apply the changes:

firebase deploy --only functions
Done! Your Help Desk can now send emails via SendGrid.

5. Set Up Inbound Email (Optional)

If you want customers to reply to tickets via email, you need to set up SendGrid's Inbound Parse.

Add MX Record

1 Add an MX record to your domain's DNS:

Type Name Priority Value
MX mail (or your subdomain) 10 mx.sendgrid.net

Configure Inbound Parse

2 In SendGrid, go to Settings → Inbound Parse

3 Click Add Host & URL

4 Configure:

  • Receiving Domain: mail.yourdomain.com
  • Destination URL: Your Firebase function URL: 
    https://us-central1-YOUR-PROJECT-ID.cloudfunctions.net/sendgridWebhook
  • Check: POST the raw, full MIME message

5 Click Add

Set Webhook Verification Key (Optional but Recommended)

6 For added security, set a webhook verification key:

firebase functions:secrets:set SENDGRID_WEBHOOK_VERIFICATION_KEY

Enter a strong random string when prompted.

6. Testing

Test Outbound Email

1 Create a new ticket in your Help Desk

2 Check that notification emails are sent to the configured admin email

Test Inbound Email (if configured)

1 Reply to a ticket notification email

2 Check that the reply appears in the ticket within a few seconds

Check Logs

If emails aren't working, check the Firebase Functions logs:

firebase functions:log --only sendgridWebhook

Troubleshooting

Emails Not Sending

Problem Solution
API key error Verify the API key is correct: firebase functions:secrets:access SENDGRID_API_KEY
Domain not verified Check domain verification status in SendGrid dashboard
Emails going to spam Complete domain authentication (DKIM, SPF records)
Rate limit exceeded Check your SendGrid plan limits; upgrade if needed

Inbound Emails Not Working

Problem Solution
MX record not found Verify MX record with: dig MX mail.yourdomain.com
Webhook not receiving Check the Destination URL is correct in Inbound Parse settings
Parsing errors Check Firebase function logs for error messages
Need Help? Check the SendGrid Documentation or contact your system administrator.
